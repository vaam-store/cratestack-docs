# CrateStack

CrateStack is a schema-first Rust framework for teams that want one source of truth for PostgreSQL-backed services instead of hand-wiring delegates, routes, auth checks, and client contracts.

<Info>
Define your service once in `.cstack`, then generate typed SQLx-backed delegates, Axum routes, policy-aware server behavior, and Rust or Dart client surfaces.
</Info>

## Why CrateStack

CrateStack is built for teams that want framework leverage without giving up Rust ownership. You define the data model and policy surface once, and the framework handles the repetitive service plumbing around it.

<Tabs>
  <Tab title="What you author">
    You write a `.cstack` schema that defines models, procedures, auth shape, mixins, and policy rules.
  </Tab>
  <Tab title="What gets generated">
    CrateStack can generate SQLx-backed delegates, Axum CRUD and procedure routes, policy-aware runtime surfaces, and typed Rust or Dart clients.
  </Tab>
  <Tab title="Where it fits best">
    The current slice is strongest for internal Rust services that want typed CRUD, generated contracts, and host-owned auth without building a custom framework from scratch.
  </Tab>
</Tabs>

## From Schema To Service

<Steps>
  <Step title="Define the contract">
    Model your auth principal, shared mixins, models, and policies in a `.cstack` file.
  </Step>
  <Step title="Compile the runtime">
    Use `include_schema!` to validate the schema at compile time and generate the `cratestack_schema` module.
  </Step>
  <Step title="Wire your database and host auth">
    Build the generated runtime with a PostgreSQL pool and bind your application auth through the host-owned provider boundary.
  </Step>
  <Step title="Choose your surface area">
    Use the generated delegates directly, mount generated Axum routers, or do both depending on how much API surface you want exposed.
  </Step>
</Steps>

## What You Get Today

1. compile-time schema validation
2. generated SQLx-backed model delegates
3. generated Axum CRUD and procedure routes
4. generated policy enforcement for the supported subset
5. generated Rust and Dart client surfaces
6. reusable schema mixins for shared field sets
7. banking-grade primitives: idempotency, optimistic locking, transactional audit, explicit transaction isolation, soft delete, rate limiting, validators, Decimal scalar, forward-only migrations

<Warning>
CrateStack is usable today, but the public `v0` track is narrower than the broader target architecture. COSE transport, parts of the client runtime, richer schema coverage, and some policy features are still deferred. Use [Current State](/overview/current-state) as the source of truth for adoption decisions.
</Warning>
